# JWST Project Rules

These rules apply to this JWST mapping and plotting project.

## Hard visualization rule

- No AI-generated images.
- All visual outputs must be generated by Python code from explicit data, catalog tables, cosmology calculations, or archive metadata.
- Use Matplotlib or other Python plotting libraries only.
- Do not use image-generation tools for scientific plots, galaxy maps, redshift maps, distance maps, or any JWST/Hubble visualization.

## Data handling

- Prefer metadata/catalog/cosmology plotting before downloading large FITS files.
- Do not download FITS/image products unless explicitly requested.
- If a dataset is curated or approximate, label it clearly in the code output and notes.
- If a coordinate is approximate or field-level, flag it in the CSV and comments.

## Colab workflow

- Prefer tiny runner cells using curl plus %run.
- Do not use %load unless explicitly requested, because it expands long code into the notebook.
- Keep the notebook clean and avoid large pasted code blocks.

## Output convention

- Print CODE OUTPUT with version ID.
- Save CSV outputs under /content/JWST_OUTPUT/CSV.
- Save plot outputs under /content/JWST_OUTPUT/PNG.
- Print an OUTPUT SUMMARY with generated paths.
- Next-to-last printed line should be the local timestamp.
- Last printed line should be the version tag.
